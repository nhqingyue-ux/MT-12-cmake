## 一、注意事项

### 安全警告
1. 如果本产品的故障或异常可能导致系统重大事故，请在外部设置适当保护电路
2. 完成所有接线前不要通电
3. 请在规定型号范围内使用
4. 请不要用在易燃、易爆气体场所
5. 请不要触摸电源端子等高压部分
6. 请不要分解、修理或改造本产品

### 使用须知
1. 本产品为 A 级仪器
2. 进行了强化绝缘防触电保护
3. 输入输出信号线超过 30 米时需设置抑制噪声突波电路
4. 定期维修是必要的

|----------|------|
| 全时 PID | 1°C 以上 P 控制，0.5°C 以上 I 控制 |

## 三、应用程序更新

1. 将温度板拨码开关 S1 拨至 ON
2. 将 `M3Temp.bin` 放至 USB 设备主目录
3. 安装至 MX1 控制 CPU 板 USB 串口
4. 从 HMI 将 MMI 地址 35550 的 BIT0 设为 ON

**LED 状态指示：**
- 下载准备中：LD1 灭、LD2 亮
- 下载进行中：LD1 灭、LD2 灭
- 下载完成：LD1 亮、LD2 灭（失败则 LD2 亮）

## 四、温度控制说明

### PID 模式

| 参数 | 说明 |
|------|------|
| P 值（比例带） | 设定越小功率越大（0.1°C 单位） |
| I 值（积分时间） | 以 6 秒积分效应控制（0.01 秒单位） |
| D 值（微分时间） | 以 60 秒微分效应控制（0.01 秒单位） |

**特殊情况：**
- P=0 → 恒温 ON/OFF 控制
- I=0 → PD 控制
- D=0 → PI 控制

### 单次自动调谐
- 常温下（35°C 以下）启动
- 第 1 次加热 5 分钟或到达设定值后停止
- 等待功能状态为 OFF 即完成

### 同步升温
以单块模组（12 段）判断温升效应做温度控制调谐

### 二段加热
1. 设定另一个温度值做加热设定值
2. 设定加热时间
3. 到达后进入标准加热设定值

### 警告状态
- 高低温偏差、断线、自动调谐中、加热状态、PWM 百分比、感测异常（30 分钟振荡未超过 3°C）

## 五、故障分析

### 通信故障
1. 检查 LD1 是否亮起
2. 检查配线、插头接触
3. 检查波特率（38400）
4. 检查 ID 设定（0x00~0x07）
5. 数据长度不可为奇数、零或大于 255 字节

### 硬件故障
1. 检查电源 LED（LD3、LD4）
2. 检查 LD1 闪烁频率
3. 检查过负载 LED（LD7）
4. 检查 LD8~LD19 状态
5. 温度漂移振荡 → 检查接地
6. 检查 K/J 型选择

### 软件故障
1. 检查温度设定、PID 值
2. 检查加热功能开关
3. 检查接触器模式和周期时间
4. RS485 通信异常时系统强制停止输出

## 六、通讯协议

### 基本参数
- 接口：RS485
- 波特率：38400，8N1
- ID：0x00~0x07（拨码开关）

### 配线注意
- RS232 保证距离 10m，RS485 保证距离 15m（建议长距离使用 RS485）
- 机壳（FG）务必接地
- SSR 负极接控制卡输出端 X3 第 13PIN GND
- **勿接 24V POWER GND**（否则过电流保护失效，短路可能损坏控制卡）

### 命令 0x30 — 多笔不连续地址读取

**请求格式：**
```
[ID:1B] [0x30:1B] [LenHi:1B] [LenLo:1B] [Addr1Hi:1B] [Addr1Lo:1B] ... [CRCLo:1B] [CRCHi:1B]
```

**响应格式：**
```
[ID:1B] [0x30:1B] [Len:1B] [Val1Hi:1B] [Val1Lo:1B] ... [CRCLo:1B] [CRCHi:1B]
```

- Length = 地址数 × 2（字节数），必须为偶数
- 最大 256 字节

### 命令 0x03 — 多笔连续地址读取

**请求格式：**
```
[ID:1B] [0x03:1B] [StartAddrHi:1B] [StartAddrLo:1B] [LenHi:1B] [LenLo:1B] [CRCLo:1B] [CRCHi:1B]
```

**响应格式：**
```
[ID:1B] [0x03:1B] [Len:1B] [Val1Hi:1B] [Val1Lo:1B] ... [CRCLo:1B] [CRCHi:1B]
```

### 命令 0x60 — 多笔不连续地址写入

**请求格式：**
```
[ID:1B] [0x60:1B] [LenHi:1B] [LenLo:1B] [Addr1Hi:1B] [Addr1Lo:1B] [Val1Hi:1B] [Val1Lo:1B] ... [CRCLo:1B] [CRCHi:1B]
```

**响应格式：**
```
[ID:1B] [0x60:1B] [Len:1B] [Addr1Hi:1B] [Addr1Lo:1B] [Val1Hi:1B] [Val1Lo:1B] ... [CRCLo:1B] [CRCHi:1B]
```

### 命令 0x06 — 单笔地址写入

**请求格式：**
```
[ID:1B] [0x06:1B] [AddrHi:1B] [AddrLo:1B] [ValHi:1B] [ValLo:1B] [CRCLo:1B] [CRCHi:1B]
```

**响应格式：** 同请求格式

### 命令 0xF0 — 异常响应

```
[ID:1B] [0xF0:1B] [ErrHi:1B] [ErrLo:1B] [CRCLo:1B] [CRCHi:1B]
```

| 错误码 | 含义 |
|--------|------|
| 0x81 | 数据长度异常 |
| 0x82 | 数据内容异常 |
| 0x83 | CRC 计算异常 |

## 七、CRC16 计算

标准 CRC16 查表法：
- 初始值：CRCHi = 0xFF, CRCLo = 0xFF
- 索引：`wIndex = CRCLo ^ byte`
- 更新：`CRCLo = CRCHi ^ CRCHiTable[wIndex]`，`CRCHi = CRCLoTable[wIndex]`
- 返回：`(CRCHi << 8) | CRCLo`
- **传输顺序：CRCLo 先发，CRCHi 后发**

```c
static unsigned int CRC16(unsigned char *data, unsigned char dataLen)
{
    BYTE chCRCHi = 0xFF;
    BYTE chCRCLo = 0xFF;
    WORD wIndex;
    while (dataLen--)
    {
        wIndex = chCRCLo ^ *data++;
        chCRCLo = chCRCHi ^ chCRCHTalbe[wIndex];
        chCRCHi = chCRCLTalbe[wIndex];
    }
    return ((chCRCHi << 8) | chCRCLo);
}
```
